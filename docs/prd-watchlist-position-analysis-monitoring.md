## 1. 背景

当前自选页已经具备股票查看能力，页面中包含顶部业务页签、分组筛选、收益概览、自选列表、设置入口、添加自选入口和底部导航。

从现有页面结构看，顶部已有 `自选 / 持仓 / 触发 / 模拟` 四个业务入口。但在真实用户场景中，用户打开自选页更多是为了盘中观察、跟踪持仓、发现异动，并对重点股票设置提醒。`模拟` 对当前自选页的主任务支撑较弱，容易让用户误解为交易演练或策略工具。

本期将顶部业务页签调整为：

- `自选`
- `持仓`
- `触发`
- `盯盘`

其中 `盯盘` 用于记录和管理用户已经设置盯盘的股票信息，包括股票、盯盘类型、触发条件、状态、有效期和最近触发情况。

## 2. 目标

让用户在自选页内完成“看自选、看持仓、设盯盘、看触发”的闭环，降低盘中人工盯盘成本，并让重点股票的状态、风险和提醒规则更清晰。

## 3. 问题描述

当前自选页主要解决“我关注的股票现在怎么样”，但没有完整解决“哪些股票需要重点盯”和“盯盘规则在哪里管理”。

具体问题：

- 自选列表与持仓信息割裂，用户难以快速识别哪些自选股已经持仓。
- 盯盘能力没有明确一级入口，用户不知道在哪里查看和管理已设置的盯盘股票。
- 触发提醒只能表达“发生了什么”，但缺少与盯盘规则的对应关系。
- 原顶部 `模拟` 页签与自选页主任务关联弱，不适合作为本期核心入口。
- 如果只在股票行中展示 `盯盘中` 标签，功能可见性不足，不利于需求评审和真实使用。

## 4. 目标用户

### 4.1 主要用户

有自选列表且存在股票持仓的交易用户。

### 4.2 次要用户

暂未持仓，但会通过自选页持续观察股票机会，并希望为重点股票设置提醒的用户。

## 5. 用户场景

### 5.1 盘中查看自选

用户打开自选页，想快速查看关注股票的涨跌、持仓标识、是否已经设置盯盘，以及是否存在触发提醒。

### 5.2 设置重点股票盯盘

用户发现某只股票接近关键价格或出现波动，希望为其设置价格、涨跌幅、异动或消息盯盘，减少人工盯盘成本。

### 5.3 查看已设置盯盘

用户想知道自己当前盯了哪些股票、每只股票设置了什么条件、是否生效、什么时候到期、最近是否触发过。

### 5.4 处理触发提醒

股票达到盯盘条件后，用户进入 `触发` 页签查看原因，并选择查看详情、修改盯盘、稍后提醒或关闭本次提醒。

## 6. 当前页面事实

基于 Figma Page 1 的 `自选` 页面，当前页面包含以下结构：

- 顶部状态栏。
- 顶部业务页签：`自选 / 持仓 / 触发 / 模拟`。
- 右侧今日股票收益入口。
- 分组筛选区域。
- 收益概览，如平均涨幅、加自选后表现。
- 添加自选入口。
- 股票列表表头。
- 自选股票列表。
- 设置入口。
- 底部导航。

本需求默认基于以上结构做增量优化，不重构整页。顶部业务页签中的 `模拟` 需要替换为 `盯盘`。

## 7. 范围

### 7.1 In Scope

- 将顶部业务页签调整为 `自选 / 持仓 / 触发 / 盯盘`。
- 在自选列表中识别并标记已持仓股票。
- 在自选页增加轻量持仓分析摘要。
- 支持从自选列表和持仓分析中设置盯盘。
- 支持价格、涨跌幅、异动、消息四类盯盘条件。
- 新增 `盯盘` 页签，展示用户已设置盯盘的股票与规则。
- 完善 `触发` 页签，展示今日或近期已触发的盯盘记录。
- 支持对盯盘规则进行查看、编辑、暂停、恢复和删除。
- 支持触发提醒后的处理动作：查看详情、修改盯盘、稍后提醒、关闭本次提醒。
- 补充空状态、加载状态、异常状态和权限提示。

### 7.2 Out Of Scope

- 不做真实下单、撤单、交易确认。
- 不做自动交易执行。
- 不做策略回测、模拟交易或交易演练。
- 不保留顶部 `模拟` 页签作为本期入口。
- 不做完整资产分析页。
- 不做复杂策略编辑器。
- 不做跨券商账户同步。
- 不改变 App 底部导航结构。
- 不改变现有自选页的主视觉风格。

## 8. 页面目标与信息架构

### 8.1 页面目标

让自选页从单纯行情列表，升级为支持持仓识别、盯盘设置、盯盘管理和触发处理的交易辅助页面。

### 8.2 顶部业务页签

| 页签 | 定位 | 主要内容 |
| --- | --- | --- |
| 自选 | 查看关注股票 | 分组、收益概览、持仓摘要、自选股票列表、盯盘状态 |
| 持仓 | 查看持仓分析 | 持仓总览、收益贡献、风险提示、建议动作 |
| 触发 | 查看已触发提醒 | 触发列表、触发原因、处理动作 |
| 盯盘 | 管理盯盘股票 | 已设置盯盘股票、规则、状态、有效期、最近触发 |

### 8.3 主任务

用户进入自选页后，可以快速完成：

- 判断哪些自选股已经持仓。
- 判断当前持仓风险是否需要关注。
- 为重点股票设置盯盘。
- 查看已设置盯盘的股票和规则。
- 查看并处理盯盘触发提醒。

## 9. 功能需求

### 9.1 顶部业务页签调整

#### 用户意图

用户需要在自选页内快速进入与当前任务相关的模块，而不是进入与盘中盯盘关系较弱的模拟模块。

#### 系统行为

- 顶部业务页签由 `自选 / 持仓 / 触发 / 模拟` 调整为 `自选 / 持仓 / 触发 / 盯盘`。
- 默认进入 `自选` 页签。
- `触发` 页签存在未处理提醒时，应展示轻量红点或数量提示。
- `盯盘` 页签存在即将到期、暂停或异常规则时，应展示轻量状态提示。

#### 显示要求

- 页签数量保持 4 个，不新增第二行页签。
- 视觉样式沿用当前页面顶部业务页签。
- `盯盘` 文案必须直接显示，不使用图标替代文字。

### 9.2 自选列表增加持仓与盯盘识别

#### 用户意图

用户希望在自选列表中直接看出哪些股票已持仓、哪些股票已设置盯盘、哪些股票有触发提醒。

#### 系统行为

- 系统根据用户持仓数据匹配自选列表股票。
- 已持仓股票在股票行内展示 `持仓` 标签。
- 已设置盯盘股票展示 `盯盘中` 状态。
- 当日有触发提醒的股票展示触发标识，如 `已触发`、`急拉触发`、`跌破成本线`。
- 股票行的更多操作中提供 `设置盯盘` 或 `管理盯盘`。

#### 展示内容

- 股票名称。
- 股票代码。
- 最新价。
- 涨跌幅。
- 持仓标签。
- 盯盘状态。
- 触发标识。

#### 交互

- 点击股票行：进入个股详情。
- 点击 `持仓` 标签：进入该股票的持仓分析。
- 点击 `盯盘中` 标签：进入该股票的盯盘规则详情或打开盯盘管理面板。
- 点击触发标识：进入该股票的触发记录详情。
- 点击更多操作：展示操作菜单，包含设置盯盘、编辑分组、移除自选。

### 9.3 自选页持仓分析摘要

#### 用户意图

用户希望进入自选页后，快速了解今日持仓整体表现和风险状态。

#### 系统行为

在自选页当前收益概览附近增加一个轻量持仓分析摘要模块。

#### 展示内容

- 持仓股票数。
- 今日持仓盈亏。
- 风险等级。
- 待处理建议数。

#### 示例文案

- `持仓 8 只`
- `今日盈亏 +233.12`
- `风险偏高`
- `2 项待处理`

#### 交互

- 点击摘要模块：切换到 `持仓` 页签。
- 点击待处理建议：进入持仓页签中的建议区。

### 9.4 持仓页签

#### 用户意图

用户希望集中查看持仓表现、风险原因和可执行的观察动作。

#### 系统行为

点击顶部 `持仓` 页签后，页面展示持仓分析内容。

#### 必须包含模块

1. 持仓总览
   - 今日盈亏。
   - 持仓市值。
   - 可用仓位。
   - 综合风险等级。

2. 收益贡献
   - 贡献最高股票。
   - 拖累最大股票。
   - 对组合收益的影响说明。

3. 风险提示
   - 单股仓位过高。
   - 行业集中度偏高。
   - 短线波动放大。
   - 盘中异动风险。

4. 建议动作
   - 继续观察。
   - 设置盯盘。
   - 查看个股详情。
   - 查看相关触发。

#### 限制

- 建议动作只能作为辅助判断，不得表达为真实交易指令。
- 不出现 `买入`、`卖出`、`减仓`、`补仓` 等直接交易动作按钮。

### 9.5 触发页签

#### 用户意图

用户希望集中查看今天哪些股票触发了盯盘条件，以及为什么触发。

#### 系统行为

点击顶部 `触发` 页签后，页面展示盯盘触发列表。

#### 展示内容

- 股票名称。
- 触发时间。
- 触发类型。
- 触发条件。
- 当前价格。
- 当前涨跌幅。
- 触发原因。
- 对应盯盘规则。
- 建议处理动作。

#### 触发类型

- 价格触发。
- 涨跌幅触发。
- 异动触发。
- 消息触发。

#### 处理动作

- 查看详情。
- 修改盯盘。
- 稍后提醒。
- 关闭本次提醒。

#### 与盯盘页签的关系

- `触发` 页签展示已经发生的提醒记录。
- `盯盘` 页签展示用户正在管理的盯盘规则。
- 同一只股票可以同时出现在 `触发` 和 `盯盘` 页签中。

### 9.6 盯盘页签

#### 用户意图

用户希望集中查看和管理自己已经设置盯盘的股票，知道每只股票盯的是什么、是否生效、最近有没有触发。

#### 系统行为

点击顶部 `盯盘` 页签后，页面展示用户已设置盯盘的股票列表和规则摘要。

#### 必须包含模块

1. 盯盘概览
   - 正在盯盘股票数。
   - 今日已触发数量。
   - 即将到期规则数量。
   - 暂停或异常规则数量。

2. 盯盘股票列表
   - 股票名称。
   - 股票代码。
   - 当前价格。
   - 当前涨跌幅。
   - 盯盘类型。
   - 触发条件。
   - 规则状态。
   - 有效期。
   - 最近触发时间。

3. 快捷筛选
   - 全部。
   - 价格。
   - 涨跌幅。
   - 异动。
   - 消息。

4. 规则操作
   - 查看规则。
   - 编辑规则。
   - 暂停盯盘。
   - 恢复盯盘。
   - 删除规则。

#### 规则状态

| 状态 | 含义 | 页面表现 |
| --- | --- | --- |
| 生效中 | 规则正常运行 | 展示 `生效中` |
| 已触发 | 今日或近期达到条件 | 展示 `已触发`，可跳转触发详情 |
| 已暂停 | 用户手动暂停 | 展示 `已暂停`，提供恢复操作 |
| 已过期 | 规则超过有效期 | 展示 `已过期`，提供重新设置 |
| 异常 | 行情、权限或服务异常导致不可用 | 展示异常提示 |

#### 交互

- 点击盯盘股票行：查看该股票的盯盘规则详情。
- 点击 `编辑`：打开盯盘设置面板并带入已有条件。
- 点击 `暂停`：规则停止触发，但保留在列表中。
- 点击 `恢复`：规则重新生效。
- 点击 `删除`：二次确认后删除规则。
- 点击 `已触发`：切换到 `触发` 页签并定位到对应记录。

### 9.7 设置盯盘

#### 用户意图

用户希望为重点股票设置提醒条件，减少人工盯盘成本。

#### 入口

- 自选列表股票行更多操作。
- 自选列表中的 `盯盘中` 标签。
- 持仓页签中的建议动作。
- 盯盘页签中的新增入口。
- 触发页签空状态中的设置入口。

#### 设置项

- 选择股票。
- 选择盯盘类型。
- 设置触发条件。
- 设置提醒方式。
- 设置有效期。
- 设置是否重复提醒。

#### 盯盘类型与条件

| 类型 | 条件示例 |
| --- | --- |
| 价格盯盘 | 高于某价格、低于某价格 |
| 涨跌幅盯盘 | 当日涨幅超过 5%、当日跌幅超过 3% |
| 异动盯盘 | 放量、急拉、急跌、封板、开板 |
| 消息盯盘 | 公告、新闻、研报、行业事件 |

#### 默认推荐

- 已持仓股票默认推荐开启风险盯盘。
- 高波动股票推荐开启异动盯盘。
- 用户可以关闭推荐，不强制开启。

#### 设置成功后

- 股票行展示 `盯盘中`。
- `盯盘` 页签新增或更新对应股票规则。
- 如果存在相同股票和相同条件，系统提示已有相同规则。

## 10. 页面状态

### 10.1 默认状态

- 默认进入 `自选` 页签。
- 展示持仓分析摘要。
- 展示自选股票列表。
- 已持仓股票展示 `持仓` 标签。
- 已设置盯盘股票展示 `盯盘中` 状态。

### 10.2 无自选状态

- 展示空状态文案：`暂无自选股票`。
- 提供 `添加自选` 入口。

### 10.3 无持仓状态

- `持仓` 页签展示空状态文案：`暂无持仓，可先关注自选股表现`。
- 自选页持仓摘要展示无持仓状态，不展示错误数据。

### 10.4 无触发状态

- `触发` 页签展示空状态文案：`今日暂无盯盘触发`。
- 提供 `去盯盘设置` 入口，点击后进入 `盯盘` 页签或打开设置面板。

### 10.5 无盯盘状态

- `盯盘` 页签展示空状态文案：`暂无盯盘股票`。
- 提供 `添加盯盘` 入口。
- 可推荐从自选股票或持仓股票中选择。

### 10.6 行情数据异常

- 保留列表结构。
- 异常字段展示 `--`。
- 页面顶部或列表区域展示更新时间异常提示。
- 已设置盯盘规则不删除，只展示异常状态。

### 10.7 通知权限关闭

- 用户设置盯盘时，如果系统通知权限关闭，展示提示。
- 用户仍可在 App 内查看触发记录。
- 不阻断规则创建，但需要提示 App 外提醒可能不可用。

## 11. 交互流程

### 11.1 查看自选股票盯盘状态

1. 用户进入自选页。
2. 页面默认展示 `自选` 页签。
3. 用户在股票列表中看到 `持仓`、`盯盘中` 或触发标识。
4. 用户点击 `盯盘中` 标签。
5. 系统打开该股票的盯盘规则详情，或进入 `盯盘` 页签并定位该股票。

### 11.2 设置盯盘

1. 用户在自选列表点击股票更多操作。
2. 用户选择 `设置盯盘`。
3. 系统打开盯盘设置面板。
4. 用户选择盯盘类型和触发条件。
5. 用户确认后，股票行展示 `盯盘中` 状态。
6. `盯盘` 页签中新增对应股票规则。

### 11.3 管理盯盘规则

1. 用户点击顶部 `盯盘` 页签。
2. 页面展示已设置盯盘的股票列表。
3. 用户选择某只股票的规则。
4. 用户可以查看、编辑、暂停、恢复或删除规则。
5. 操作完成后，列表状态同步更新。

### 11.4 处理触发提醒

1. 股票达到盯盘条件。
2. 系统在 `触发` 页签生成一条触发记录。
3. 用户进入 `触发` 页签。
4. 用户查看触发原因和对应盯盘规则。
5. 用户选择查看详情、修改盯盘、稍后提醒或关闭本次提醒。

### 11.5 查看持仓分析并设置盯盘

1. 用户点击顶部 `持仓` 页签。
2. 页面展示持仓总览、收益贡献、风险提示和建议动作。
3. 用户点击某条建议中的 `设置盯盘`。
4. 系统打开盯盘设置面板，并预填股票与推荐条件。
5. 用户确认后，规则进入 `盯盘` 页签管理。

## 12. 设计与原型要求

- 原型以 390px 移动端为准。
- 默认生成亮色模式。
- 保留当前自选页的顶部结构、列表密度和底部导航。
- 顶部业务页签必须为 `自选 / 持仓 / 触发 / 盯盘`。
- 不做通用 AI 聊天页。
- 不做营销式大卡片。
- 不做桌面展示说明区。
- 信息密度保持接近当前自选页。
- 新增模块优先使用现有卡片、列表行、标签、按钮、底部弹层。
- 金融涨跌颜色语义保持一致。
- 所有交易相关文案必须表达为观察、提醒、辅助判断，不得写成真实交易指令。
- `盯盘` 必须有明确可见入口，不能只隐藏在股票标签或弹层中。

## 14. 验收标准

### 14.1 顶部业务页签

- Given 用户进入自选页，when 页面加载完成，then 顶部业务页签展示 `自选 / 持仓 / 触发 / 盯盘`。
- Given 旧版本存在 `模拟` 页签，when 本需求上线，then 顶部业务页签不再展示 `模拟`。
- Given 用户点击 `盯盘`，when 页面切换完成，then 展示已设置盯盘的股票和规则列表。

### 14.2 自选列表

- Given 自选股票存在持仓，when 用户查看自选列表，then 股票行展示 `持仓` 标签。
- Given 股票已设置盯盘，when 用户查看自选列表，then 股票行展示 `盯盘中` 状态。
- Given 股票当日触发提醒，when 用户查看自选列表，then 股票行展示触发标识。
- Given 用户点击 `盯盘中`，when 股票存在盯盘规则，then 系统展示该股票的盯盘规则详情或定位到盯盘页签中的对应记录。

### 14.3 持仓分析摘要

- Given 用户存在持仓，when 用户进入自选页，then 页面展示持仓股票数、今日盈亏、风险等级和待处理建议数。
- Given 用户点击持仓摘要，when 页面切换完成，then 进入 `持仓` 页签。
- Given 用户无持仓，when 页面加载完成，then 摘要区域展示无持仓状态，不展示错误数据。

### 14.4 持仓页签

- Given 用户点击 `持仓`，when 页面切换完成，then 页面展示持仓总览、收益贡献、风险提示和建议动作。
- Given 页面展示建议动作，then 动作文案不得包含直接交易指令。
- Given 用户点击 `设置盯盘`，when 设置面板打开，then 面板预填对应股票。

### 14.5 触发页签

- Given 存在触发记录，when 用户进入 `触发` 页签，then 每条记录包含股票、时间、类型、条件、原因、对应规则和处理动作。
- Given 无触发记录，when 用户进入 `触发` 页签，then 页面展示 `今日暂无盯盘触发`。
- Given 用户点击 `修改盯盘`，when 设置面板打开，then 面板带入对应规则。
- Given 用户关闭单条触发提醒，when 操作完成，then 该提醒从未处理列表移除或状态变为已关闭。

### 14.6 盯盘页签

- Given 用户存在盯盘规则，when 用户进入 `盯盘` 页签，then 页面展示盯盘概览和盯盘股票列表。
- Given 用户不存在盯盘规则，when 用户进入 `盯盘` 页签，then 页面展示 `暂无盯盘股票` 和 `添加盯盘` 入口。
- Given 用户点击某只股票的盯盘规则，when 详情打开，then 页面展示盯盘类型、触发条件、提醒方式、有效期和规则状态。
- Given 用户暂停规则，when 操作完成，then 规则状态变为 `已暂停`。
- Given 用户恢复规则，when 操作完成，then 规则状态变为 `生效中`。
- Given 用户删除规则，when 二次确认后完成，then 规则从盯盘列表中移除。

### 14.7 设置盯盘

- Given 用户选择有效股票和有效条件，when 点击确认，then 系统创建或更新盯盘规则。
- Given 创建成功，when 用户返回自选列表，then 对应股票展示 `盯盘中`。
- Given 创建成功，when 用户进入 `盯盘` 页签，then 对应股票出现在盯盘列表中。
- Given 用户设置相同股票和相同条件，when 点击确认，then 系统提示已有相同规则。
- Given 通知权限关闭，when 用户设置盯盘，then 系统提示 App 外提醒可能不可用，但不阻断 App 内规则创建。

## 15. 依赖与风险

### 15.1 依赖

- 账户持仓数据。
- 自选列表数据。
- 行情数据。
- 盯盘规则服务。
- 触发记录服务。
- 消息或异动数据源。
- 通知权限能力。

### 15.2 风险

- 持仓分析涉及交易辅助，需要确认合规文案边界。
- 盯盘条件过多可能导致设置流程复杂，需要优先保证常用条件。
- 行情数据延迟可能影响触发准确性。
- 用户可能把提醒理解为交易建议，页面文案需要明确“仅作观察提醒”。
- 如果盯盘页签信息密度过高，可能影响 390px 移动端可读性。

## 16. 待确认问题

- `触发` 页签展示范围是仅展示今日触发，还是展示近 7 日触发？
- `盯盘` 页签是否允许同一股票设置多条规则？
- 盯盘规则默认有效期是当日有效、长期有效，还是用户必选？
- 通知方式是否包含 App 外推送、App 内消息、短信或其他渠道？
- 持仓风险等级由后端计算返回，还是前端基于字段展示？
- 合规是否要求在持仓分析和盯盘设置区域统一展示免责声明？
