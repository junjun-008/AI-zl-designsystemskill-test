# stock-trading-design-system 中文说明

## 这个 Skill 是什么

`stock-trading-design-system` 是股票交易 App 的设计系统规范 Skill。

它的核心作用是约束页面和组件输出，让新页面、新模块或原型尽量符合当前 App 的视觉语言、信息密度和组件规则。

## 这次新增了什么

新增了 `assets/icon-index.json`，用于管理图标和 Logo 的业务语义；新增 `assets/icons/`，用于保存可以被 HTML 原型直接调用的通用 SVG 图标资产。

它解决的问题是：后续做原型或页面时，不只知道“这里需要一个图标”，还要知道“这个图标应该代表什么业务含义”。

## 图标索引现在覆盖什么

- 导航图标：返回、菜单、底部导航。
- 通用操作图标：添加、设置、复制、筛选。
- 自选相关图标：加自选、自选分组、排序。
- 持仓相关图标：持仓、可用资金。
- 盯盘相关图标：盯盘、已触发、规则生效、规则暂停。
- 行情相关图标：上涨趋势、消息。
- Logo 相关：股票标识、银行标识。

## 使用原则

- 涉及图标或 Logo 时，优先读取 `assets/icon-index.json`。
- 不要因为图标形状相似就随便替换，要看业务语义是否匹配。
- 股票 Logo 不允许用空圆形或随机图标代替。
- 如果真实 Figma 图标暂时拿不到，可以使用索引里的 fallback，但需要标记为待替换。
- 通用功能图标优先通过 `assets/icons/icon-map.json` 调用，不在页面里重新手写 SVG。
- 特殊页面图标可以临时绘制，但必须标记为临时图标，不得覆盖通用图标资产。

## 通用 Icon 资产

- `assets/icons/icon-map.json`：图标 ID、中文含义、分类和状态。
- `assets/icons/common-icons/icon-sprite.svg`：通用 SVG 图标 Sprite。
- HTML 原型生成时，将 Sprite 内联到页面，避免本地 `file://` 预览时外部 SVG 引用失效。

## 在当前项目中的定位

它是设计系统约束层。它告诉后续原型生成：页面应该长什么样、组件应该怎么组织、图标应该表达什么含义，哪些视觉规则不能随意改变。
