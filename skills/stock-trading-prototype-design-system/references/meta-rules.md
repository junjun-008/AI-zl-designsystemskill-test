# 规则治理

## MR-01 来源优先级

新增或修改规则前，必须确认来源。优先级：

1. 用户或设计负责人确认。
2. 结构化 token 和资产。
3. Page 1 真实源稿。
4. 重复出现且根因明确的评审案例。
5. 单次反馈只能作为观察项。

## MR-02 新增规则流程

1. 确认现有规则没有覆盖。
2. 找到来源。
3. 判断规则类型：token 事实、硬规则、带原因的判断规则、案例。
4. 写入最窄的相关文件。
5. 更新 `CHANGELOG.md`。
6. 如果容易回归，加入 eval。

## MR-03 冲突处理

如果规则冲突：

```text
用户确认 > 结构化资产 > Page 1 源稿 > 详细 reference > SKILL.md 摘要 > 模型推断
```

不能静默选择。需要标注冲突并请求确认。

## MR-04 案例不自动变规则

一次失败不等于规则。只有当根因明确、来源可信、可泛化、可检查时，才把案例升级为规则。
