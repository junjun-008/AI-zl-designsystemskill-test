# 页面模式规则

结构化来源见 `assets/page-patterns.json`。本文件只写生成原型时需要执行的规则。

## 1. 行情首页类 `market-home`

适用：

- 已有行情首页修改。
- 市场概览。
- 行情异动提醒。
- 首页新增模块。

推荐结构：

```text
顶部状态/导航
市场或工作区导航
市场摘要
指数/股票数据卡片
实时简报或资讯模块
快捷入口
排行/主题/股票列表
底部导航
```

规则：

- 保持 390px 移动端高信息密度。
- 不使用桌面大屏 KPI 仪表盘风格。
- 金融数据必须使用 rise/fall token。
- 新增模块应插入到合适的卡片流中，不打破页面壳。

## 2. AI 交易/交易辅助类 `ai-trading`

适用：

- AI 持仓诊断。
- 交易优化建议。
- AI 交易员。
- 自动模拟交易。
- 交易计划生成。

推荐结构：

```text
AI 交易头部
说明文案
当前持仓/选股区域
优化方案卡片
时间线或事件行
主操作按钮
历史记录或次级入口
```

规则：

- 不做成通用聊天机器人页面。
- 必须保留金融交易场景和风险语义。
- 主操作按钮要明确动作含义，不使用模糊文案。
- 如涉及交易执行，必须在原型说明中标注数据假设和非真实交易。

## 3. 详情/事件信息类 `detail-event`

适用：

- 事件详情。
- 发展脉络。
- 热点事件。
- 关联 ETF。
- 市场信息解释。

推荐结构：

```text
单行页面标题
摘要模块
时间线/关系列表
相关股票或 ETF 卡片
解释文本
必要时加入底部操作
```

规则：

- 优先保证信息结构和阅读顺序。
- 不使用营销式大标题。
- 解释文本和数据卡片要有清晰层级。

## 4. 已有页面修改类 `page-modification`

适用：

- 已有页面修改。
- 功能调整与添加。
- 新增模块功能。
- 新增风险提示、状态提示、操作区。

流程：

1. 先识别最接近的已有页面模式。
2. 保留原页面壳、导航和整体节奏。
3. 只新增或替换目标模块。
4. 在原型说明中写清楚修改点。
5. 需要交互时，只补足评审必须理解的状态。

规则：

- 默认不重构整页。
- 新增模块必须使用现有卡片、数据行、按钮、提示区的视觉语言。
