# 视觉规则

精确值来自 `assets/design-tokens.css`。不要在规则文件里复制大量数值。

## Token 使用

- 颜色、字号、圆角、间距、阴影优先使用 CSS 变量。
- 不直接写未经验证的 raw hex 作为新语义。
- 新增模块应复用已有语义 token，而不是创造新颜色。

## 金融语义

- 上涨、正向、增强：使用 `--color-finance-rise`。
- 下跌、负向、减弱：使用 `--color-finance-fall`。
- 排名热度：优先使用 rank token。
- 不根据个人习惯交换红绿语义。

## 禁止风格

- 不使用营销页大 Hero。
- 不使用通用 SaaS 仪表盘。
- 不使用紫色渐变、大面积装饰光斑、无关插画。
- 不使用与 Page 1 风格无关的大圆角卡片系统。

## 字号层级

常用层级：

- 11px：辅助数字或极小说明。
- 12px：元信息。
- 13px：导航/Tab。
- 14px：正文和常规数据标签。
- 18px：重点数字。
- 22px：顶部标题或关键标题。

## 缺失素材处理

如果图标、图片或真实组件没有来源：

1. 使用低调占位。
2. 明确标注“待替换”。
3. 不把占位当成最终规范。
