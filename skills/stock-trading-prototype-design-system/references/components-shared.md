# 共享模块与组件模式

结构化来源见 `assets/component-patterns.json`。

## 组件选择原则

- 先使用和需求最接近的 Page 1 模块模式。
- 原型里可以用 HTML 结构模拟模块，但视觉必须使用 token。
- 如果真实图标缺失，使用简化占位并标注“图标待替换”。
- 不使用通用 SaaS 卡片、营销 CTA 卡片、桌面仪表盘组件替代 App 模块。
- 已有页面修改时，先从截图或 Figma 节点识别现有组件形态，再决定是否复用本文件规则。
- 本文件只沉淀跨页面组件规则，不写死具体页面模板。

## MVP 模块

### 移动端页面壳

用于所有 MVP 原型。默认 390px 宽，移动端纵向布局。

### 市场指数卡片

用于市场概览、行情首页、行情异动。

必须包含：

- 指数名。
- 指数值。
- 涨跌幅。
- rise/fall 语义。

### 实时简报卡片

用于行情首页、异动提醒、资讯摘要。

必须保持紧凑行结构，不要写成长文章布局。

### 快捷入口网格

用于首页或功能入口。

规则：

- 图标 + 标签居中。
- 标签使用真实业务动作。
- 缺失图标必须标注。

### 底部导航

用于需要 App 主导航的页面。

规则：

- 不发明新的导航结构。
- 当前 Tab 使用 active 语义。
- HTML 原型中可以固定在底部。
- 如果来源截图里底部导航是圆角浮层、玻璃态或带客服/顾问浮标，原型应保留这种结构关系。
- 缺少真实 icon 时使用一致的低调占位；不得把占位当作最终图标。

### 顶部业务 Tab

用于页面内主业务切换，例如自选、持仓、触发、模拟等。

规则：

- 位置、字号、选中态下划线或高亮必须优先参考来源截图。
- 新增功能不得随意改变 Tab 顺序。
- 有提醒状态时使用小红点或来源截图已有提醒样式。
- 如果用户只要求修改一个 Tab 内容，其他 Tab 壳结构应保留。

### 胶囊筛选

用于分组、分类、时间范围、状态筛选。

规则：

- 高度、圆角、横向间距优先跟随来源截图。
- 选中态只改变必要的文字色、背景、边框或底部光效。
- 胶囊数量超出屏宽时保持横向滚动或裁切节奏，不改成多行按钮组。

### 股票列表行

用于自选、行情、持仓、相关股票等列表。

必须包含或保留来源中的核心字段：

- 股票名称。
- 股票代码和市场标签。
- 最新价。
- 涨跌幅或涨跌额。
- 必要时保留分时小图、信号标签、更多操作。

规则：

- 列表是高密度信息区，不改成大卡片瀑布流。
- 数字列必须右对齐。
- 行高、分割线和二级标签密度必须贴近来源截图。
- 金融涨跌颜色使用 `--color-finance-rise` 和 `--color-finance-fall`。
- 新增状态如持仓、盯盘中、触发，应以小标签或小红点融入列表行，不额外插入大模块。

### 股票状态标签

用于持仓、盯盘中、触发、信号、市场属性、风险状态。

规则：

- 标签高度应紧凑，通常服务于列表行或摘要区。
- 标签文案必须短，优先 2-5 个中文字符。
- 状态标签不能抢过股票名称、价格和涨跌幅。
- 风险或触发标签可以使用红/橙语义，但要避免大面积警告块。

### 分时小图 / Mini Sparkline

用于列表行中展示短周期走势。

规则：

- 来源截图中存在分时小图时，HTML 原型应保留同等占位和尺寸关系。
- 可用简化 SVG/polyline 表达走势，但必须保持小尺寸和低噪声。
- 不扩展为大图表，不加入坐标轴、网格线或仪表盘样式。

### 底部弹层

用于设置盯盘、确认模拟、补充操作。

规则：

- 从屏幕底部出现，圆角只在顶部。
- 遮罩必须覆盖当前手机画布，不影响画布外区域。
- 操作按钮数量控制在 1-2 个主按钮，复杂选项用紧凑宫格或列表。
- 交易相关动作必须表达为模拟、提醒或辅助判断，不写成真实下单确认。

### Toast / 轻提示

用于设置成功、关闭提醒、加入观察等短反馈。

规则：

- 文案短，1.5 秒左右消失。
- 不替代正式错误处理或重要确认。
- 不遮挡底部导航和核心操作。

### 指标行

用于紧凑 label-value 数据。

### 进度条

用于比例、进度、强弱程度表达。

### AI 交易员头部

用于 AI 交易/交易辅助页面。不能做成普通聊天助手。

### 股票推荐卡片

用于 AI 交易和选股场景。必须体现股票名、价格、表现、状态。

### 时间线事件行

用于 AI 交易事件、市场事件、详情页发展脉络。

### 风险/状态提示模块

用于页面修改类需求中新增风险提示、状态提示、异常提示。

规则：

- 融入当前卡片节奏。
- 不使用 Web 管理后台 alert 风格。
