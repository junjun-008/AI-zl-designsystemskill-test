# 术语表

用于统一业务、设计、代码和原型生成中的命名。面向用户的解释以中文为主，文件名和资产字段保留英文。

## 核心术语

| 推荐术语 | 含义 | 文件/代码术语 | 备注 |
|---|---|---|---|
| 原型 | 可用于评审的页面初稿 | `prototype` | MVP 默认是 HTML 可交互原型 |
| 页面模式 | 一类页面的结构模板 | `page pattern` | 如行情首页、AI 交易页、详情页 |
| 模块 | 页面内可复用业务区块 | `module` / `component pattern` | 不等同于 Figma 组件实例 |
| 视觉 token | 颜色、字号、间距、圆角、阴影等数值 | `design token` | 精确值来自 `assets/design-tokens.css` |
| 行情首页 | 市场概览和首页信息聚合页面 | `market-home` | 来源：Page 1 首页类框架 |
| AI 交易 | AI 交易员、交易辅助、优化建议等页面 | `ai-trading` | 不能做成通用聊天机器人风格 |
| 详情/事件 | 事件、脉络、关联标的等信息解释页 | `detail-event` | 信息层级优先 |
| 页面修改 | 在已有页面中新增或调整模块 | `page-modification` | 默认不重构整页 |
| 上涨/正向 | 金融数据正向变化 | `rise` | 使用 `--color-finance-rise` |
| 下跌/负向 | 金融数据负向变化 | `fall` | 使用 `--color-finance-fall` |

## 禁用或容易混淆的说法

| 避免使用 | 应使用 | 原因 |
|---|---|---|
| 高端感、科技感、现代感 | 具体 token、布局、组件规则 | 形容词不可验证 |
| 大屏仪表盘 | 移动端行情/交易页面 | 当前 Skill 面向 390px App 原型 |
| 营销页 | 原型页面 | MVP 不生成营销落地页 |
| 聊天助手页面 | AI 交易/交易辅助页面 | 当前 AI 场景是金融交易辅助，不是通用聊天 |
| 随便加个图标 | 图标待替换或使用来源图标 | MVP 图标库尚未完成 |

## 维护规则

当生成结果里出现新的业务词、同义词或误用词，先更新本文件，再决定是否新增规则。
