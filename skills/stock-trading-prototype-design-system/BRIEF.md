# 股票交易 App 原型设计系统 Skill MVP 简报

## 1. 目标

创建一个可复用的 Codex Skill，帮助产品经理和设计师快速生成符合当前股票交易 App 视觉风格的原型页面。

这个 Skill 的核心任务是：把一个新业务需求，或者一个页面修改需求，转成适合需求评审、业务沟通、领导汇报、早期设计测试的移动端原型初稿。

MVP 阶段的目标质量：

- 生成 70-85 分的高保真原型初稿，整体视觉要像当前 App。
- 优先保证评审可用、产品流程清晰、视觉风格一致，不追求一次生成设计师最终精修稿。
- 支持设计师在生成结果基础上继续细化，而不是替代设计师最终设计。

## 2. 主要用户

### 产品经理

使用这个 Skill 来：

- 把新需求快速转成可评审的移动端原型页面。
- 向业务方或领导说明页面结构、核心信息层级和关键交互。
- 在正式进入设计前，先探索页面修改方案。

### 设计师

使用这个 Skill 来：

- 生成符合当前 App 视觉语言的页面初稿。
- 测试布局方向、模块组合、明暗色效果和交互想法。
- 在方向确认后，继续手动优化细节。

- 源页面：`Page 1`

当前已经整理出来的设计系统页面和组件页，只作为参考材料。只有当某个组件或规则被重新验证、重建，并且能对应到 `Page 1` 的真实源稿后，才能作为正式规范。

规则优先级：

1. Figma `Page 1` 中真实、已认可的页面。
2. 已确认的 token 和明暗色映射。
3. 从源稿重建并验证过的组件或模块模式。
4. 用户确认过的评审问题和修正案例。
5. 模型推断只能作为假设，必须明确标注。

## 4. MVP 覆盖范围

MVP 阶段优先支持这些页面和原型场景：

1. 行情首页类页面
   - 市场指数概览。
   - 实时资讯/简报。
   - 快捷入口。
   - 股票和指数数据卡片。

2. AI 交易和交易辅助类页面
   - AI 交易员概览。
   - 交易优化建议。
   - 股票推荐卡片。
   - 时间线/事件行。
   - 操作按钮和状态提示。

3. 详情或事件信息类页面
   - 事件详情。
   - 个股相关详情。
   - 时间线或相关市场信息。
   - 数据摘要和说明模块。

4. 页面修改类需求
   - 在已有页面模式里新增模块。
   - 调整某个信息区块。
   - 新增风险提示、状态提示、操作区域，同时保持当前 App 风格。

## 5. MVP 不覆盖的内容

MVP 阶段不做这些事：

- 不生成营销落地页、品牌活动页、海报或通用网页。
- 不替代 PRD 写作 Skill。
- 不承诺一次生成设计师最终精修稿。
- 不发明当前 App 之外的新视觉语言。
- 不发明没有来源的颜色、字体、阴影、图标或导航样式。
- 不承诺所有输出都是完整 Figma 组件实例。
- 不在第一版覆盖所有 App 页面类型。

## 6. 输出形式

MVP 优先输出：

1. HTML 可交互原型
   - 生成和修改速度快。
   - 适合内部评审和交互说明。
   - 视觉必须遵循从 `Page 1` 提取的 App 风格。
   - MVP 阶段以 HTML 原型图为主，不优先写入 Figma。

2. 原型说明摘要
   - 页面目的。
   - 主要模块。
   - 数据假设。
   - 交互状态。
   - 设计规则自检。

后续输出：

- 当页面模式、token 和组件规则稳定后，再支持 Figma 原型生成。

## 7. Skill 必须具备的能力

这个 Skill 必须帮助模型做到：

- 把需求路由到正确的页面模式。
- 选择当前 App 风格里的模块，而不是通用 UI 模块。
- 使用已确认的颜色、字体、圆角、间距、阴影 token。
- 保持股票/金融语义，尤其是涨跌红绿语义。
- 在有源稿依据时支持明暗色映射。
- 基于当前 App 的 390px 移动端页面风格生成布局。
- 在需要时包含评审可用的交互，例如 Tab 切换、选中态、弹窗、页面跳转、加载、空状态、错误状态。
- 当源稿证据不足时，明确说明假设。
- 当用户没有指定主题时，默认生成亮色模式。

## 8. MVP 硬规则

- 如果 `Page 1` 里已有对应模式，不允许凭空发明视觉样式。
- 不允许把占位图标当作真实 App 图标使用。
- 不允许改变金融涨跌红绿语义。
- 不允许使用通用 SaaS 仪表盘风格、营销页大 Hero 风格、装饰性渐变风格。
- 不允许使用大圆角卡片、紫色渐变、无关插画风格，除非源稿明确支持。
- 当前整理出来的 Components 页不能直接当作权威组件库，必须逐个验证或重建。
- 每次生成原型后，都必须附带简短自检。

## 9. 第一批评测任务

MVP Skill 需要用这些任务测试：

1. “生成一个 AI 持仓诊断页面，展示风险等级、持仓建议、可执行操作，风格要符合当前 App。”
2. “新增一个行情异动提醒页面，包含指数变化、异动原因、相关股票和快捷操作。”
3. “做一个暗色模式 AI 交易优化页，展示当前持仓、优化方案和执行按钮。”
4. “在现有行情首页风格里新增一个风险提示模块，用于提示用户今日市场波动较大。”
5. “帮我写一篇营销海报文案。”预期行为：这个 Skill 不应该触发。
